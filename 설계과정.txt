>>keyboard 설명<<

  키보드의 입출력을 담당하는 모듈이다. def on_key_event(event) 함수는 키보드가 눌렸을 때 작동하는 함수이다. 
event.event_type은 2가지 속성을 가진다. down은 키보드를 누름을, up은 키보드를 땜을 의미한다. 
위 내용은 키보드의 입력을 감지하는 명령들이었고, 이후 출력 방법을 서술한다.
  keyboard.write("내용")은 괄호 안 문자열을 키보드로 치는 함수이다. 
keyboard.press('enter')는 엔터를 누르는, keyboard.release('enter')는 엔터를 때는 함수로, 해당 프로젝트에서 주로 사용할 두 함수이다.
keyboard.on_press(on_key_press)는 키보드 입력을 감지하겠다는, 예를들면 cctv의 전원이다. on.press는 클릭했을때, hook는 up, down 상태일 때 함수를 실행한다.

일단 기초적인 틀을 잡기 위해 '프로그램 실행 후 키보드 입력을 받고, esc를 누르면 프로그램 실행부터 esc를 누른 시점 까이의 기록을 출력하는 프로그램'을 완벽하게 짜는것에 초점을 맞췄다.

<1차매크로 - 1 commit>
키보드의 down 상태를 저장할 time_click_down와 키보드의 up 상태를 저장할 time_click_up 까지 총 2개의 딕셔너리를 만들어준다.
해당 딕셔너리의 keys 에는 키보드를 누르거나 뗀 시간이 저장되고, values 에는 무슨 버튼을 눌렀는지가 저장이 된다.
시간 함수 time은 유닉스 에포크를 사용하기 때문에 프로그램 시작 시간에 초기화 되지 않는다. 즉, 스톱워치나 타이머를 사용하기 위해서는 time함수와 관련한 연산이 기본적이다.
time함수는 소수 자리가 매우 긴 float형을 반환하기 때문에 time과 관련된 연산은 round(<해당 연산>,3)으로 소숫점 3번째 자리에서 끊어주었다. 
이와 관련해 esc가 눌렸다면 해당 시간과 프로그램 시작 시간의 차이를 구해 키보드 입력이 몇 초 동안 이루어졌는지 저장하는 stopwatch 변수를 만들어준다.
esc가 눌린 시간에 start_time2 변수에 해당 시간을 저장해 stopwatch 만큼의 타이머를 설계해준다.
녹화 시간 만큼 키보드 버튼을 눌러주는 코드를 실행하기 위해 while문을 사용해 주었다.
반복문 내에서 esc를 누른 후 경과한 시간이 키보드를 눌렀을 때의 시간과 같다면 해당 keys값의 values 이름을 가진 키보드 자판을 눌러주는 코드를 작성했다.

문제점
1. (while round(time.time()-start_time2,3) != stopwatch: 에서 stopwatch가 round(...)와 일치하지 않아) 무한루프에 빠질 수 있음.
2. (반복문이 실행되는 속도는 워낙 빠르고 시간을 소숫점 3번째 자리까지만 사용하기 때문에) 같은 시간동안 반복문이 여러번 실행되어 하나의 자판을 여러번 클릭 할 수 있음.
3. down 상태일때만 반응하기 때문에 출력은 자판을 누르고 방치하게 된다.

<2차매크로 - 2 commit>
 기존 문제점 3가지 중 첫번째 무한루프를 해결해 보자.
해당 문제점은 프로그램 실행 중 시간이 소수점 부분에서 건너뛰게 될 가능성이 있기 때문에 발생한다.
따라서 두 시간의 동이(!=) 비교가 아닌 대소(>=) 비교를 통해 해당 문제를 해결 할 수 있다.
 두번째 문제는 하나의 자판이 여러번 클릭되는 문제이다.
해당 문제는 같은시간 동안 반복문이 2번 이상 실행되기 때문에 발생한다.
따라서 이미 실행한 시간대는 딕셔너리 내에서 삭제해줘야 반복문이 동일한 시간에 대한 실행을 2번 이상 반복하지 않는다.
 세번째 문제는 up 상태의 부재로 인한 자판의 반복이다.
두번째 문제와 비슷한 문제이지만 up상태가 없어 프로그램은 사용자가 해당 자판을 계속 누르고 있었다고 생각할 가능성이 있다.
따라서 up 상태 또한 추가해 사용자의 입력을 그대로 출력할 수 있도록 했다.

문제점
1. 프로그램이 자판을 누를 때 사용자가 입력한 키가 가끔 안눌리는 현상 발생
 - 해당 문제의 이유는 esc를 누르고 타이머의 시간이 소수점 부분에서 불연속되기 때문이다.(딕셔너리에 기록되어 있던 시간을 타이머가 넘겨버려 자판이 씹히는 현상)
 여기에서 불연속은 0.001 -> 0.003 같이 연속된 숫자의 흐름이 아니기 때문이다.

<for실험1 - if_for_test brunch>
 기존 코드를 저장하고 언제든지 백업하기 위해 새로운 브런치를 하나 생성 후 해당 문제를 해결해 보기로 한다.
if문으로는 지나가는 시간을 비교할 때 허술한 부분이 생겼다.
때문에 딕셔너리 안에 있는 시간을 직접적으로 비교할 수 있는 for문으로 해당 문제를 해결하고자 했다.
for문으로 딕셔너리 안에 있는 keys 값인 사용자 입력 시간들을 하나씩 받아와 해당 시간을 현재 타이머와 비교한다.
비교할 때 서로가 같음을 비교하는게 아닌, 현재 시간이 딕셔너리에서 받아온 시간보다 크거나 같은 경우로 비교해 넘어간 시간 또한 상정한다.
현재 시간이 딕셔너리에서 받아온 시간보다 크다는 것은 딕셔너리에서 이미 실행된 시간은 삭제 되었을 텐데 아직 남아있는 경우에 대한 것이다.
예를 들어, 딕셔너리에서 받은 시간은 0.001, 0.002, 0.003 ... 이고, esc를 누른 후 실행되는 타이머는 0.001, 0.003 ... 을 지난다고 해보자.
이 때, 현재 타이머는 0.002를 건너뛰기 때문에 0.002에 대한 while문은 자동으로 넘어간다. 
따라서 반복문이 0.003을 실행할 때 딕셔너리 시간은 이전에 실행되고 삭제된 0.001 제외 0.002, 0.003이 남게 된다.
이때 for문이 리스트(keys 값 리스트로 변환) 내에서 0.002를 가져왔다면 for문 안 if문은 현재 시간 0.003이 더 크기 때문에 지나간 시간을 실행시키고 딕셔너리 내에서 삭제한다.
이를 통해 해당 문제를 해결했다.

문제점
1. 동시입력시 (딕셔너리의 특성상 같은 keys가 다른 values를 가질 수 없기 때문에) 하나의 키가 눌리지 않는 현상
